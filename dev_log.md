# HanPDF Editor 개발 및 리팩토링 로그 (Dev Log)

이 문서는 프로젝트의 개발 이력, 알려진 이슈, 그리고 향후 아키텍처 리팩토링 및 기능 고도화 계획을 기록하는 공식 로그 파일입니다.

---

## 📌 현재 버전 상태: v1.1.0 (2차 완성본)
* **주요 기능 완료:** 
  * 로컬 PDF 로드 및 고해상도(150 DPI) 캔버스 렌더링
  * 마우스 드래그를 통한 White-out 및 텍스트 오버레이 편집
  * **미리보기 & Nudge:** 방향키 1픽셀 이동, +/- 폰트 크기 조절
  * **스마트 스포이드:** 드래그 영역 잉크 색상 자동 추출 및 `C` 키 수동 사이클 기능
  * **굵기 조절:** `B` 키를 통한 맑은 고딕 볼드체 토글 기능

---

## ✅ 완료된 과제 1: 아키텍처 리팩토링 (v1.1.1 반영 완료)
코드의 안정성, 가독성, 유지보수성을 극대화하기 위해 기획된 5가지 리팩토링이 **모두 성공적으로 완료**되었습니다.

### 1. [완료] `main_window.py` 렌더 책임 분리
* **해결 내용:** 마우스 이벤트와 미리보기/Nudge 로직을 전담하는 `ui/canvas_editor.py` 클래스를 신설하여 코드를 분리했습니다. 이로써 `main_window.py`는 순수 레이아웃과 렌더링에만 집중합니다.

### 2. [완료] 관심사 분리(SoC) 완벽 준수
* **해결 내용:** `PDFService` 내부에 `apply_overlay`, `save_pdf` 등 Facade 패턴을 적용하여, UI 레이어에서 백엔드 `fitz.Document` 객체에 직접 접근하는 의존성을 완벽히 제거했습니다.

### 3. [완료] 비동기 스레딩(Threading) 도입
* **해결 내용:** PDF 로드 및 고해상도 이미지 변환(`get_page_image`) 작업을 `threading.Thread`로 분리하여 백그라운드에서 처리하고, 완료 시 `root.after()`를 통해 UI 캔버스를 안전하게 업데이트하도록 개선했습니다. UI Freezing이 완벽히 해결되었습니다.

### 4. [완료] PyInstaller 배포 대비 리소스 경로 래핑
* **해결 내용:** `sys._MEIPASS`를 활용하는 `utils/resource.py` 유틸리티를 신설하고, `editor_service.py` 폰트 로드 시 절대 경로 래핑 및 시스템 폰트 Fallback 방어 로직을 적용했습니다.

### 5. [완료] Undo/Redo 아키텍처 데이터 모델 확장
* **해결 내용:** `models/app_state.py` 내부에 `action_history`와 `redo_history` 스택 자료구조를 미리 선언하여, 향후 실행 취소 기능 추가를 위한 데이터 뼈대를 구축했습니다.
