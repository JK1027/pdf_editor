# 요구사항 정의서 (PRD): HanPDF Editor (Architecture Revised)

## 1. 프로젝트 개요 (Project Overview)
- **제품명 (가칭):** HanPDF Editor (선생님을 위한 로컬 PDF 간편 수정기)
- **목적:** 외부 서버 업로드 없이, 로컬 환경에서 학습지 및 개인 문서의 오타를 고치거나 문항을 수정하는 가벼운 데스크톱 프로그램.
- **개발 철학:** 
  1. **빠른 실행력 (MVP):** CustomTkinter를 활용해 눈에 보이는 1차 작동 버전을 빠르게 확보.
  2. **관심사 분리 (Separation of Concerns):** 추후 더 강력한 렌더링 프레임워크(PySide6 등)로의 마이그레이션을 완벽히 대비하기 위해, UI(화면) 레이어와 비즈니스 로직(PDF 처리) 레이어를 철저히 분리하여 설계.

## 2. 핵심 아키텍처 및 편집 전략
- **가리고 덧그리기 (Mask and Redraw):** 원본 PDF의 복잡한 텍스트/객체 속성을 직접 수정하지 않고, 사용자가 지정한 좌표에 흰색 사각형을 그려 덮은(White-out) 뒤, 그 위에 새 텍스트를 오버레이하는 방식을 채택하여 수식 및 표 레이아웃 손상을 원천 차단.
- **철저한 모듈 격리:** UI 렌더링 코드는 절대로 PDF 파일의 바이너리나 객체 좌표를 직접 다루지 않으며, 모든 PDF 제어는 `services` 폴더 내의 독립된 컨트롤러를 통해서만 이루어짐.

## 3. 기술 스택 (MVP 기준)
- **UI 프레임워크:** `CustomTkinter` (빠르고 세련된 네이티브 윈도우 UI 구성)
- **PDF 서비스 레이어:** `PyMuPDF (fitz)` (초고속 PDF 로드, 픽스맵 렌더링, 좌표 변환, 마스킹 및 텍스트 객체 삽입)
- **이미지 렌더링 처리:** `Pillow (PIL)` (PyMuPDF가 생성한 픽스맵 데이터를 CustomTkinter 캔버스에 띄우기 위한 변환)

## 4. 프로젝트 디렉토리 구조 (Strict Separation)
```text
HanPDFEditor/
│
├── main.py                 # CustomTkinter 앱 실행 진입점 (Entry Point)
├── ui/                     # UI 전용 모듈 (화면 그리기 및 이벤트 리스너)
│   ├── main_window.py      # 전체 프레임 및 중앙 캔버스 영역
│   └── toolbar.py          # 좌측 도구 모음(사이드바) 및 버튼 UI
│
├── services/               # 비즈니스 로직 (UI 프레임워크와 완전히 독립된 구역)
│   ├── pdf_service.py      # PDF 파일 I/O 및 이미지 렌더링 엔진
│   └── editor_service.py   # 캔버스 좌표 ↔ PDF 좌표 변환 및 오버레이 편집 엔진
│
└── requirements.txt        # 의존성 패키지 (customtkinter, pymupdf, pillow)
```

## 5. 4단계 개발 로드맵

### 1단계 [뼈대 세우기: UI 컴포넌트 구성]
- **목표:** 백엔드 로직 연결 없이 CustomTkinter 메인 윈도우 프레임과 레이아웃 완성.
- **구현 항목:** 
  - 좌측 툴바 (파일 열기, 뷰어/편집 모드 선택, 저장 버튼).
  - 우측 메인 영역 중앙에 PDF 이미지가 들어갈 캔버스(Canvas) 생성.
  - 하단 네비게이션 (이전/다음 페이지 버튼 및 페이지 번호 텍스트).

### 2단계 [내용 채우기 1: PDF 렌더링 및 뷰어 기능]
- **목표:** `services/pdf_service.py`를 구현하고 UI에 연결.
- **구현 항목:** 
  1. PDF 업로드 시 지정된 해상도(DPI)로 페이지를 이미지 변환. (학습지의 작은 수식 가독성 확보).
  2. 변환된 이미지를 메인 UI 캔버스에 출력.
  3. 이전/다음 버튼 이벤트 발생 시 `pdf_service`를 호출하여 화면 갱신.

### 3단계 [내용 채우기 2: 좌표 매핑 및 오버레이 로직]
- **목표:** 이 프로젝트의 핵심인 `services/editor_service.py` 구현.
- **구현 항목:**
  1. 사용자가 UI 캔버스에서 마우스 드래그로 사각형 영역을 지정하면 시작/끝 좌표(X,Y) 추출.
  2. UI의 캔버스 좌표를 실제 원본 PDF 해상도 좌표계로 보정(Mapping).
  3. 보정된 PDF 좌표에 흰색 도형(White-out)을 덧씌우고, 사용자가 입력한 새 텍스트 삽입.
  4. 변경된 상태를 `수정본_파일명.pdf`로 안전하게 저장 및 원본 보존 로직 추가.

### 4단계 [마무리 점검 및 마이그레이션 대비 검토]
- **목표:** 실사용을 위한 예외 처리 및 아키텍처 점검.
- **점검 항목:** 
  1. `ui/` 폴더 내의 코드가 `fitz` (PyMuPDF) 라이브러리를 직접 import 하는 일이 없는지 최종 리뷰.
  2. 스크롤, 줌, 윈도우 창 크기 조절 시 캔버스 내 좌표계가 틀어지는 현상 방어 로직 추가.
  3. 한글 텍스트 삽입 시 기본 폰트(맑은 고딕 등) 강제 맵핑을 통한 텍스트 깨짐 현상 예외 처리.
